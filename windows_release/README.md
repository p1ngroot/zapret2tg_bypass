# Zapret2 TUI - Telegram Unblocker для Windows

Легковесный TUI интерфейс для обхода блокировок Telegram на базе zapret2.

## 🚀 Возможности

- ✅ Обход блокировок Telegram (TLS, MTProto, HTTP)
- ✅ Легковесный TUI интерфейс на базе Textual
- ✅ Автоматическое обнаружение блокировок
- ✅ Поддержка множественных стратегий обхода
- ✅ Мониторинг статистики в реальном времени
- ✅ Гибкая конфигурация через UI
- ✅ Автохостлист для автоматического обнаружения заблокированных доменов

## 📋 Требования

- Windows 10/11 (x64)
- Python 3.8+
- WinDivert драйвер (входит в комплект winws2)
- winws2.exe (из проекта zapret2)

## 🛠️ Установка

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Скачивание winws2

Скачайте последнюю версию winws2 из релизов zapret2:
https://github.com/bol-van/zapret2/releases

Поместите `winws2.exe` и `WinDivert64.dll` в корневую директорию проекта.

### 3. Запуск

```bash
python zapret2_tui.py
```

Или используйте batch файл:

```bash
start.bat
```

## ⚙️ Конфигурация

### Стратегии обхода

По умолчанию включены 3 стратегии:

1. **TLS Fake** - Отправка фейковых TLS пакетов с плохой контрольной суммой
2. **Multisplit** - Разделение TLS handshake на части
3. **UDP Length** - Изменение длины UDP пакетов для MTProto

### Настройка портов

TCP порты (по умолчанию): `443,80,88,8080`
UDP порты (по умолчанию): `443,3478,3479,3480,3481,5222,5223,5224,8888,9898,9899,9900,10000-10100`

### Глобальные настройки

- **Отладка** - Включить подробное логирование
- **Фоновый режим** - Запуск в фоне
- **Исправление IPv4/IPv6** - Исправление маршрутизации
- **Кэш имён хостов** - Кэширование DNS для ускорения
- **Фильтр LAN/Loopback** - Фильтрация локального трафика

## 🔧 Решение проблем

### WinDivert ошибка

Если возникает ошибка WinDivert:
1. Убедитесь что у вас есть права администратора
2. Проверьте что WinDivert64.dll и WinDivert64.sys на месте
3. Перезагрузите компьютер

### Telegram не работает

1. Проверьте что служба запущена (статус "РАБОТАЕТ")
2. Включите режим отладки и проверьте логи
3. Попробуйте изменить стратегию в конфигурации
4. Проверьте что порты правильно указаны

### Ошибка "Access Denied"

Запустите от имени администратора.

## 🎯 Как это работает

1. **Перехват трафика**: WinDivert перехватывает сетевые пакеты на уровне ядра
2. **Анализ**: Lua скрипты анализируют пакеты и определяют тип трафика
3. **Модификация**: Применяются стратегии обхода DPI:
   - Fake пакеты с неправильной контрольной суммой
   - Разделение пакетов на части
   - Изменение TTL/HL
   - Модификация TCP опций
4. **Отправка**: Модифицированные пакеты отправляются дальше
